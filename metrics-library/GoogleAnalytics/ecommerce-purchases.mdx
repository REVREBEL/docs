# E-commerce Purchases Table

## Schema Purpose
The `EcommercePurchases` table houses item-level telemetry that tracks interactions across the digital purchase funnel.[1] It records detailed product metadata alongside transactional milestones, including impressions, additions to carts, checkout actions, and absolute sales revenue generated by individual product items.[1]

## Table Schema

| Column Name | Data Type | Column Definition & Calculated Metric Representation |
| :--- | :--- | :--- |
| itemId | STRING | The unique machine-readable identifier of the item.[1] |
| itemName | STRING | The human-readable name of the specific item.[1] |
| itemBrand | STRING | Brand name of the item.[1] |
| itemCategory | STRING | The primary product classification category.[1] |
| itemCategory2 | STRING | The secondary nested product classification category.[1] |
| itemCategory3 | STRING | The tertiary nested product classification category.[1] |
| itemCategory4 | STRING | The quaternary nested product classification category.[1] |
| itemCategory5 | STRING | The quinary nested product classification category representing the deepest category level.[1] |
| itemVariant | STRING | The specific variation attributes of a product, populated by the `item_variant` parameter.[1] |
| itemListPosition | INTEGER | The position of an item in a list, populated in tagging by the `index` parameter in the items array.[1] |
| itemsViewed | INTEGER | The number of units viewed for a single item, counting `view_item` events.[1] |
| itemAddedToCart | INTEGER | The quantity of units added to cart for a single item, counting `add_to_cart` events.[1] |
| itemsPurchased | INTEGER | The number of units for a single item successfully sold in `purchase` events.[1] |
| itemRevenue | FLOAT | The total revenue from purchases minus refunded transaction revenue from items only.[1] Formulated as: $$\text{Item Revenue} = \text{Item Price} \times \text{Item Quantity}$$ Excludes tax and shipping values, which are calculated at the transaction event level.[1] |

## Analytical Considerations
Unlike total revenue in other tables, `itemRevenue` is calculated strictly at the line-item level.[1] This means it excludes order-level shipping costs and taxes, reflecting only the direct product revenue.[1] 

This structure is highly useful for cart abandonment analyses, allowing teams to calculate the conversion rate of specific items from views to additions and completed purchases:

$$\text{Item Purchase Conversion Rate} = \frac{\text{itemsPurchased}}{\text{itemsViewed}}$$
