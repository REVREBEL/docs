# GA4 Origin Tables

This section documents the first round of Google Analytics 4 processed tables used by the Metrics Library. These tables originate from the `metrics_g4a` schema and are currently modeled as processed `p_ga4_*` tables tied to the GA4 property suffix `281286275`.

The current documentation focuses on table purpose, grain, source context, business use, and metric interpretation so analysts, developers, and AI agents can understand how each table should be used before wiring it into dashboards or downstream models.

## First-Round Documentation Standard

Each GA4 origin table page should include:

1. **Schema Purpose** — what the table represents and why it exists.
2. **Table Schema** — field names, data types, and business definitions.
3. **Analytical Considerations** — how the table should be interpreted and common pitfalls.
4. **Lineage Context** — whether the table is session-scoped, user-scoped, item-scoped, event-scoped, or content-scoped.
5. **Dashboard Use Cases** — the metric cards, views, or dashboard sections that should consume the table.

## Modeling Notes

GA4 data can appear deceptively simple because many tables share similar metrics such as `activeUsers`, `sessions`, `keyEvents`, and `totalRevenue`. The most important modeling distinction is the table grain. A revenue value in a user acquisition table should not be interpreted the same way as a revenue value in an ecommerce item table.

Use this rule of thumb:

- **User acquisition** answers: who first brought the user?
- **Traffic acquisition** answers: what brought this session?
- **Landing pages** answer: where did the session begin?
- **Pages and screens** answer: where did engagement happen?
- **Events** answer: what action happened?
- **Ecommerce purchases** answer: which item moved through the funnel?
- **Promotions** answer: which internal creative influenced item engagement?
- **Tech details** answer: what device or platform context shaped the experience?
- **Demographic details** answer: where and who is the audience, subject to GA4 privacy thresholds?
- **Audiences** answer: how do defined user cohorts behave?
